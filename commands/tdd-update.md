## TDD Update (Red Phase — Update)

PRD 변경이나 개발 중 요구사항 수정 시 기존 테스트를 업데이트.

### 사용법
/tdd-update <변경 사항 설명>

### 시나리오
1. **PRD 변경**: "사용자 인증 방식이 JWT에서 세션 기반으로 변경됨"
2. **설계 변경**: "API 응답 형식이 { data: ... }에서 { result: ... }으로 변경"
3. **요구사항 추가**: "관리자 역할이 추가됨 — 권한 검사 필요"
4. **요구사항 제거**: "이메일 인증 단계 제거됨"

### Flow
1. tdd-writer agent가 Update 모드로 실행
   - 기존 테스트 파일 읽기
   - 변경 사항 분석 (추가/수정/제거)
   - 영향받는 테스트만 수정 (나머지 보존)
2. 수정된 테스트 실행 → 변경된 테스트는 FAIL, 기존 테스트는 상태 유지
3. 변경 리포트: 무엇이 바뀌었고 왜 바뀌었는지
4. 구현 업데이트 진행

### 원칙
- 변경된 요구사항에 영향받는 테스트만 수정
- 무관한 테스트는 절대 건드리지 않음
- 각 변경에 주석: `// Updated: <reason>`
